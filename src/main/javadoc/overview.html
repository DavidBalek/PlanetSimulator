Planet Simulator API

This API simulates planetary motion and provides astronomical coordinate transformations.

The architecture is divided into several key components:

  • PlanetKeplerian — Represents a celestial body using its Keplerian orbital elements (semi-major axis, eccentricity, etc.).
  • PlanetHeliocentric — Represents a planet's Cartesian position in the heliocentric coordinate system at a given time.
  • PlanetRADec — Represents the position of a body in the sky, using Right Ascension and Declination coordinates.
  • KeplerianCalculator — Handles the core orbital mechanics: propagating orbits forward in time based on Kepler's laws.
  • HorizonsAPI — Communicates with the NASA Horizons service to retrieve ephemeris.
  • TimeConverter — Provides utility functions to convert between time formats (e.g., Julian Date, J2000 epoch).
  • ArgumentParser — Parses user input when running the simulation from the command line.
  • PlanetSimulator — The class where the main is.
  • ResultData - The class that orchestrates all the previously mentioned functionality - reading input, performing calculations, and displaying results.

The simulation typically follows this flow: user input is parsed.Then data from Horizons API data are loaded. KeplerianCalculator propagates the body's position to the desired time, the position is converted to heliocentric Cartesian coordinates (PlanetHeliocentric), and then further projected into sky coordinates (PlanetRADec) for observation.
